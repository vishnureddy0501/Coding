## 2. Locking Logic (Seat Reservation)

1. **User Selects Seats**
   - Keep selection only on frontend (no DB writes yet).

2. **User Clicks “Go to Payment” → Lock**
   - Call backend `/lock-seats` with selected seat IDs.
   - Backend starts a **transaction**:
     - Check if seats are `OPEN` or `LOCKED` but expired.
     - If available → set `status=LOCKED`, `lockedBy=user/session`, `lockedAt=now`.
     - Return locked seats + `expiresAt = now + 10 mins`.

3. **Frontend Countdown Timer**
   - Show 10 min timer, warn at 2 min.
   - If timer expires → redirect back to seat selection page.

---

## 3. Booking Workflow (After Lock)

- **On Payment Success**
  - Backend `/confirm-booking` verifies seats are still locked and not expired.
  - Updates `status=BOOKED`, `bookedAt=now`, creates `Bookings` record with `status=CONFIRMED`.

- **On Payment Failure/Timeout**
  -  Don't update anything in seats collection. as time passes it will be in status = BOOKED and (time-lockedAt) > 10 mins. Hence it is treated as open only.

// 5. Get API for Available Seats
async function getAvailableSeats(eventId) {
  const seats = await Seat.find({ eventId });
  const now = Date.now();

  const availableSeats = seats.filter(seat => {
    if (seat.status === 'OPEN') return true;
    if (seat.status === 'LOCKED' && now - seat.lockedAt > 10 * 60 * 1000) return true;
    return false;
  });

  return availableSeats;
}

// 6. Concurrency & Race Conditions (Lock Seats)
async function lockSeats(selectedSeats, userId) {
  const now = Date.now();

  const result = await db.seats.updateMany(
    {
      seatId: { $in: selectedSeats },
      $or: [
        { status: "OPEN" },
        { status: "LOCKED", lockedAt: { $lt: now - 10 * 60 * 1000 } }
      ]
    },
    { $set: { status: "LOCKED", lockedAt: now, lockedBy: userId } }
  );

  if (result.matchedCount !== selectedSeats.length) {
    throw new Error("Some seats are already locked or booked. Please reselect.");
  }

  return { success: true, lockedUntil: new Date(now + 10 * 60 * 1000) };
}

// 7. Edge Cases Handling
/*
1. Partial Lock Failure:
   - If not all selected seats can be locked, throw error & ask user to reselect.

2. Payment Success but Seats Expired:
   - Before confirming booking, re-check that lockedAt + 10 mins > now.

3. Multiple Devices:
   - Use lockedBy as sessionId to prevent same user from double-locking.

4. Admin Cancels Event:
   - Update all seats for that event => status: CANCELLED.
   - Update all related bookings => status: CANCELLED.
*/
