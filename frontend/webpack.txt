🏗 What is Webpack?

Webpack is a module bundler for JavaScript applications.
Think of it as a tool that takes all your code and assets (JS, CSS, images, fonts) and bundles them into something the browser can understand (usually a few JS + CSS files).

👉 Without Webpack, browsers don’t know how to handle modern things like ES Modules, TypeScript, SCSS, etc.

🔑 Why Use Webpack?

Bundle multiple files into fewer optimized files → faster loading.

Transpile modern code (ES6+, TypeScript) → older browser support.

Load assets (CSS, images, fonts) via loaders.

Optimize performance (tree-shaking, minification, code-splitting).

Dev server for live reloading.

Supports plugins for almost anything (linting, compression, env variables).

⚡ Core Concepts of Webpack
1. Entry

Starting point(s) of your app (like index.js).

Webpack builds the dependency graph from here.

// webpack.config.js
module.exports = {
  entry: "./src/index.js",
};

2. Output

Where the final bundled files go.

output: {
  filename: "bundle.js",
  path: __dirname + "/dist",
},

3. Loaders

Webpack only understands JS/JSON natively.

Loaders tell Webpack how to process other files (CSS, images, TS, etc).

Examples:

module: {
  rules: [
    { test: /\.css$/, use: ["style-loader", "css-loader"] }, // handle CSS
    { test: /\.ts$/, use: "ts-loader" }, // handle TypeScript
    { test: /\.(png|jpg)$/, use: "file-loader" }, // handle images
  ],
},

4. Plugins

Extend Webpack’s power beyond loaders.

Can optimize bundles, inject variables, generate HTML, etc.

Examples:

const HtmlWebpackPlugin = require("html-webpack-plugin");

plugins: [
  new HtmlWebpackPlugin({ template: "./src/index.html" }),
],

5. Mode

Tells Webpack whether you’re building for development or production.

mode: "development"  // or "production"


development → fast, source maps, no minification.

production → optimized, minified, tree-shaken.

6. DevServer

For local development with live reload / hot module replacement.

devServer: {
  port: 3000,
  hot: true,
  open: true,
}

🚀 Advanced Webpack Concepts
1. Code Splitting

Split code into smaller bundles → faster initial load.

optimization: {
  splitChunks: { chunks: "all" },
}

2. Tree Shaking

Remove unused code (dead code elimination).
Works automatically in production mode with ES6 imports.

3. Dynamic Imports

Load modules only when needed.

import("./math.js").then((math) => {
  console.log(math.add(2, 3));
});

4. Environment Variables

Inject variables into the app at build time.

const webpack = require("webpack");

plugins: [
  new webpack.DefinePlugin({
    "process.env.API_URL": JSON.stringify("https://api.example.com"),
  }),
],

5. Babel + Webpack

Use Babel loader to transpile modern JS → older browser support.

{
  test: /\.js$/,
  exclude: /node_modules/,
  use: {
    loader: "babel-loader",
    options: { presets: ["@babel/preset-env"] }
  }
}

6. CSS + SASS with Webpack
{
  test: /\.s[ac]ss$/i,
  use: ["style-loader", "css-loader", "sass-loader"],
}

7. Module Federation (important for micro-frontends!)

Lets multiple apps share code at runtime.

Example (host + remote):

new ModuleFederationPlugin({
  name: "host",
  remotes: {
    app2: "app2@http://localhost:3002/remoteEntry.js",
  },
});

🏢 Webpack in Production

Minify JS/CSS (TerserPlugin, css-minimizer-webpack-plugin).

Cache busting with [contenthash].

Bundle analysis with webpack-bundle-analyzer.

Environment-specific configs (webpack.dev.js, webpack.prod.js).

✅ Alternatives to Webpack

Vite (faster dev builds, uses ESBuild).

Parcel (zero-config bundler).

Turbopack (from Vercel, future of Next.js).

Rollup (great for libraries).

Webpack is still widely used in large enterprise projects, especially with React + legacy setups and for micro-frontends (Module Federation).

🔹 1. Code Splitting (Lazy Loading in React)

Instead of bundling everything in one big JS file, Webpack lets us split the code into smaller chunks. React supports this via React.lazy + Suspense.

Example: Split a component
src/LazyComponent.js

export default function LazyComponent() {
  return <h2>I was lazy loaded! ⚡️</h2>;
}

Update src/App.js
import React, { useState, lazy, Suspense } from "react";

const LazyComponent = lazy(() => import("./LazyComponent"));

export default function App() {
  const [show, setShow] = useState(false);

  return (
    <div>
      <h1>Hello Webpack + React 🚀</h1>
      <button onClick={() => setShow(!show)}>
        {show ? "Hide" : "Show"} Lazy Component
      </button>

      <Suspense fallback={<p>Loading...</p>}>
        {show && <LazyComponent />}
      </Suspense>
    </div>
  );
}


✅ Now when you click the button, Webpack will load LazyComponent only when needed → smaller initial bundle.

Check in DevTools → Network tab. You’ll see an extra chunk file being fetched dynamically.

🔹 2. Environment Variables

You often need different API URLs for dev vs production.
Webpack doesn’t read .env automatically (like Next.js does), so we use DefinePlugin.

Install dotenv
npm install dotenv --save-dev

Create .env.development
API_URL=http://localhost:4000/api

Create .env.production
API_URL=https://myapp.com/api

Update webpack.config.js

Add at the top:

const webpack = require("webpack");
const dotenv = require("dotenv");


Load env based on mode:

module.exports = (env, argv) => {
  const isProd = argv.mode === "production";

  const envFile = isProd ? ".env.production" : ".env.development";
  const envVars = dotenv.config({ path: envFile }).parsed;

  return {
    entry: "./src/index.js",
    output: {
      path: path.resolve(__dirname, "dist"),
      filename: "[name].[contenthash].js",
      clean: true,
    },
    mode: isProd ? "production" : "development",
    module: {
      rules: [
        {
          test: /\.js$/,
          exclude: /node_modules/,
          use: {
            loader: "babel-loader",
            options: {
              presets: ["@babel/preset-env", "@babel/preset-react"],
            },
          },
        },
        {
          test: /\.css$/,
          use: ["style-loader", "css-loader"],
        },
      ],
    },
    plugins: [
      new CleanWebpackPlugin(),
      new HtmlWebpackPlugin({
        template: "./public/index.html",
      }),
      new webpack.DefinePlugin({
        "process.env": JSON.stringify(envVars),
      }),
    ],
    devServer: {
      static: {
        directory: path.join(__dirname, "dist"),
      },
      port: 3000,
      hot: true,
      open: true,
    },
  };
};
